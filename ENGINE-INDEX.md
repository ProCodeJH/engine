## ⚡ Quick Start (한 명령)

```bash
# 1. Brand Kit 초기화 (한 번)
node sigma-brand-kit.mjs init ./brand-kit
# → brand-kit/images/{hero,photos,logos,icons}/ 에 자현 자산 넣기

# 2. End-to-End Pipeline (한 명령에 모든 paradigm orchestration)
node sigma-engine.mjs https://www.dusonnuri.co.kr/ \
  --brand-kit ./brand-kit \
  --target 88 \
  --deploy vercel \
  --auto
```

자동 실행:
1. Omega mirror
2. Asset audit (license truth)
3. Brand-kit swap (자현 자산 → license PASS)
4. Convergence loop (자율 88%+ 도달)
5. Auto-deploy (Vercel)
6. CERT-ENGINE-COMPLETE.md

## 🛠️ 명령 카탈로그

### 측정
```bash
node sigma-asset-audit.mjs <projDir>
node sigma-composite-score.mjs <projDir> --source-url <url>
node sigma-composite-score.mjs <projDir> --source-url <url> --strict-license  # 정직
node sigma-fleet-dashboard.mjs .
```

### Cleanroom (license 100% PASS path)
```bash
node sigma-asset-swap.mjs <projDir>             # dry-run
node sigma-asset-swap.mjs <projDir> --execute   # placeholder swap
node sigma-brand-kit.mjs init                   # 디렉토리 초기화
node sigma-brand-kit.mjs <projDir> --execute    # 자현 자산 swap
node sigma-pure-cleanroom.mjs <projDir> --source-url <url> --swap execute
```

### 자율 도달
```bash
node sigma-converge.mjs <projDir> --source-url <url> --target 90 --swap brand-kit --brand-kit ./brand-kit
```

### Deploy
```bash
node sigma-auto-deploy.mjs <projDir>            # dry-run
node sigma-auto-deploy.mjs <projDir> --execute --prod --platform vercel
```

### 통합
```bash
node sigma-engine.mjs <source-url> --brand-kit ./brand-kit --target 88 --deploy vercel --auto
```
